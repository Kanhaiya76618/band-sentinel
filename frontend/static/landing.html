The problem // on-call is the worst seat in engineering

A 3am page. A scramble to assemble a war room. ~42 minutes of mean-time-to-resolve while revenue bleeds and one tired person guesses under pressure — scale it? roll it back? fail over? — with no way to prove the fix before touching prod.

Pages don't reason

A Slack alert tells you something broke. It can't diagnose, can't argue, can't disprove a bad fix.

Linear bots overreach

Planner→executor agents happily execute the first plausible action. Nobody plays skeptic.

Fixes go in unproven

"Scale the pods" looks fine — until the leak saturates anyway. You find out in prod.

The 5-agent war room // adversarial, not a pipeline

@observer
Detect

z-score anomaly detection over a rolling baseline; correlates the spike to a deploy and opens the room.

@diagnostician
Root cause

Forms a confidence-scored hypothesis from the evidence — "memory leak from v2.3.1," not a guess.

@remediator
Propose

Proposes a fix — and revises it when the validator shoots the first one down.

@validator
Disprove

Runs a chaos replay and holds a veto. Rejects fixes that still breach SLO.

@commander
Gate & execute

Gates the one irreversible action on a human, executes, and files the postmortem.

@observerSEV1 on checkout-api/us-east-1 — p99 z=578, right after deploy v2.3.1. Opening the room.
@diagnosticianRoot cause: memory leak from v2.3.1 (conf 86%). p99 tracks memory, not traffic.
@remediatorFix #1: scale 6 → 12 pods.
@validatorREJECTEDChaos replay still breaches: p99 8000ms, errors 12.8%. Leak is unbounded — pods won't hold.
@remediatorRevised: roll back v2.3.1 and fail over to us-west-2.
@validatorPASSEDReplay clean: p99 308ms, 0% errors, within SLO. Safe to execute with approval.
@commanderIrreversible → asks @human → approved → executes → RESOLVED. Postmortem filed.
The reject-then-fix beat is the differentiator. One agent disproves another's fix mid-incident, with evidence it generated itself — a chaos replay. That argument is a thing a notification channel physically cannot do. The reject is computed, not scripted: change the scenario and the verdict changes.

How it's built // Band is the coordination layer, not a channel

Agents never talk directly. Every inter-agent message is a RoomMessage posted to a shared AgentBus. Flip one config value and the exact same agent code runs in a real Band room — so every collaboration beat physically happens inside Band.

⬡ Band

The shared agent room. Per-agent identities post into one chat over the band-sdk REST API; the orchestrator drives the cascade through Band, reproducibly. BUS=band.

✦ Featherless AI

Powers @diagnostician & @validator (the skeptics). Open-model inference behind the CrewAI agents.

✦ AI/ML API

Powers @observer, @remediator & @commander behind the LangGraph agents.

Cross-framework (LangGraph ⇄ CrewAI ⇄ orchestrator) and cross-provider (AI/ML API + Featherless) by design — five agents, two frameworks, two model providers, one Band room.
